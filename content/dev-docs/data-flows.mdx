## Emergency Requests

### Creating an Emergency Request

1. User fills out the form in `src/components/DashboardPage.tsx`
2. Client POSTs to `src/app/api/requests/route.ts`
3. API validates auth session, checks PayPal subscription status via `src/lib/paypal-subscriptions.ts`
4. Prisma creates the `Request` record in PostgreSQL
5. Push notifications sent to nearby farm owners via `src/app/api/requests/route.ts` → `notify()`
6. Confirmation email sent via `src/lib/email.ts` → Resend API

```mermaid
sequenceDiagram
    actor Requester
    actor NearbyFarmer as Nearby Farmer
    participant Browser
    participant API as API Routes
    participant DB as PostgreSQL
    participant Push as Web Push
    participant Email as Resend Email

    Requester->>Browser: Click "New Request" in the navigation bar
    Browser->>Browser: Fill form: disaster type, farm, gate (optional), description
    Browser->>API: POST /api/requests
    API->>DB: Request.create(disasterType, farmId, userId, gateId, description)
    DB-->>API: New request
    API->>DB: Farm.findMany() within 100 miles
    DB-->>API: Nearby farms with users
    loop Each nearby farmer
        API->>Push: sendNotification("Farm X needs assistance for {type}")
        Push-->>NearbyFarmer: Push notification
    end
    API->>Email: sendRequestCreatedEmail(requester)
    Email-->>Requester: Confirmation email
    API-->>Browser: 201 Created
    Browser-->>Requester: Dashboard centers on new request
```

### Viewing Requests

Requests are fetched based on map bounds and filtered by visibility rules depending on user role and sidebar tab.

```mermaid
sequenceDiagram
    actor User
    participant Browser
    participant API as API Routes
    participant DB as PostgreSQL

    User->>Browser: Navigate to Dashboard
    Browser->>API: GET /api/requests?ne_lat=..&sw_lng=..
    API->>DB: Request.findMany(where: bounds + visibility rules)
    Note right of API: ADMIN: all requests<br/>Yours tab: own requests<br/>Members tab: org farm requests<br/>Nearby tab: all in map bounds
    DB-->>API: Requests with farm, user, responses
    API-->>Browser: Request list + map markers
    Browser-->>User: Display request markers on map
```

### Responding to a Request

1. User views request details in `src/components/DashboardPage.tsx`
2. Client POSTs response to `src/app/api/requests/[requestId]/respond/route.ts`
3. Request owner notified by email via `src/lib/email.ts`

```mermaid
sequenceDiagram
    actor Responder
    actor Requester
    participant Browser
    participant API as API Routes
    participant DB as PostgreSQL
    participant Email as Resend Email

    Responder->>Browser: Click request → "Respond to Request"
    Browser->>API: POST /api/requests/{id}/respond
    Note right of Browser: Body: ETA, equipment (optional)
    API->>DB: Validate: request not closed, no duplicate response
    API->>DB: Response.create(userId, requestId, ETA, equipment)
    DB-->>API: New response
    API->>Email: sendEmail(requester, responder details + ETA)
    Email-->>Requester: Email with responder details and ETA
    API-->>Browser: 201 Created
```

### Cancelling a Response

```mermaid
sequenceDiagram
    actor Responder
    actor Requester
```

### Closing a Request

```mermaid
sequenceDiagram
    actor Requester
    actor Responder
    participant Browser
    participant API as API Routes
    participant DB as PostgreSQL
    participant Email as Resend Email

    Requester->>Browser: Click "Close Request" → Confirm
    Browser->>API: PATCH /api/requests?requestId={id}
    Note right of Browser: Body: { action: "close" }
    API->>DB: Request.update(closedOn: now())
    DB-->>API: Updated request
    API->>Email: sendRequestClosedEmail(requester)
    Email-->>Requester: Confirmation email
    loop Each responder
        API->>Email: sendRequestClosedToResponderEmail(responder)
        Email-->>Responder: "Assistance is no longer needed"
    end
    API-->>Browser: 200 OK
```

## Organizations


### Admin: Approve or Reject Organization

```mermaid
sequenceDiagram
    actor Admin
    actor User as Requester
    participant Browser
    participant API as API Routes
    participant DB as PostgreSQL
    participant Email as Resend Email

    Admin->>Browser: Navigate to Orgs → "Org Requests" tab
    Browser->>API: GET /api/org-requests
    API->>DB: OrgRequest.findMany(status: PENDING)
    DB-->>API: Pending requests
    API-->>Browser: List of requests

    alt Approve
        Admin->>Browser: Click "Approve" + optional review notes
        Browser->>API: PATCH /api/org-requests
        Note right of Browser: { requestId, action: "approve" }
        API->>DB: BEGIN TRANSACTION
        API->>DB: Organization.create(name, desc, members: [requester as OWNER])
        API->>DB: OrgRequest.update(status: APPROVED, reviewerId, reviewedAt)
        API->>DB: COMMIT
        API->>Email: sendOrgApprovalEmail(requester, orgId)
        Email-->>User: "Your organization has been approved!"
    else Reject
        Admin->>Browser: Click "Reject" + optional review notes
        Browser->>API: PATCH /api/org-requests
        Note right of Browser: { requestId, action: "reject", reviewNotes }
        API->>DB: OrgRequest.update(status: REJECTED, reviewerId, reviewNotes)
        API->>Email: sendOrgRejectionEmail(requester, reason)
        Email-->>User: "Your organization request was rejected"
    end
    API-->>Browser: 200 OK
```

### Joining an Organization

```mermaid
sequenceDiagram
    actor Joiner as Join Requester
    actor Owner as Org Owner/Manager
    participant Browser
    participant API as API Routes
    participant DB as PostgreSQL
    participant Email as Resend Email

    Joiner->>Browser: Navigate to Orgs → "Join Organization" tab
    Browser->>Browser: Search for organization by name
    Joiner->>Browser: Click "Request to Join" → optional message → "Send Request"
    Browser->>API: POST /api/organizations/{orgId}/join-requests
    Note right of Browser: Body: { message? }
    API->>DB: Validate: org exists, not already member, no pending request
    API->>DB: OrgJoinRequest.create(userId, orgId, message, status: PENDING)
    DB-->>API: New join request
    API->>Email: sendJoinRequestSubmittedEmail(joiner)
    Email-->>Joiner: Confirmation email
    API-->>Browser: 201 Created

    Owner->>Browser: Dashboard sidebar → "Join Requests" (red badge)
    Browser->>API: GET /api/organizations/{orgId}/join-requests
    API-->>Browser: List of pending requests

    alt Approve Join
        Owner->>Browser: Click "Approve"
        Browser->>API: PATCH /api/organizations/{orgId}/join-requests
        Note right of Browser: { requestId, action: "approve" }
        API->>DB: OrganizationMember.create(userId, orgId, role: MEMBER)
        API->>DB: OrgJoinRequest.update(status: APPROVED)
        API-->>Browser: 200 OK
    else Reject Join
        Owner->>Browser: Click "Reject"
        Browser->>API: PATCH /api/organizations/{orgId}/join-requests
        Note right of Browser: { requestId, action: "reject" }
        API->>DB: OrgJoinRequest.update(status: REJECTED)
        API-->>Browser: 200 OK
    end
```

### Farm CRUD

```mermaid
sequenceDiagram
    actor User
    participant Browser
    participant API as API Routes
    participant DB as PostgreSQL

    alt Create Farm
        User->>Browser: Click dropdown → "Add Farm" → fill details
        Browser->>API: POST /api/farms
        Note right of Browser: { name, address, city, state,<br/>zip, lat, lng, acreage, year, userId }
        API->>DB: Farm.create(data)
        DB-->>API: New farm
    else Update Farm
        User->>Browser: Edit farm details
        Browser->>API: PATCH /api/farms?farmId={id}
        API->>DB: Farm.update(where: id, data: fields)
        DB-->>API: Updated farm
    else Delete Farm
        User->>Browser: Select farm → "Delete Farm" → confirm
        Browser->>API: DELETE /api/farms?farmId={id}
        API->>DB: Farm.delete(id) — cascades all relations
        DB-->>API: Deleted
    end
    API-->>Browser: Success response
```
